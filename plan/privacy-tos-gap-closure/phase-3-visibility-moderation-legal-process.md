# Phase 3: Visibility, Moderation, and Legal Process

**Status:** Not Started  
**Owner:** Product + API + Moderation Ops  
**Depends On:** Phase 1

## Goal
Eliminate product/policy mismatches around public visibility and establish minimal legal-process runbooks.

## Deliverables
1. Public/private visibility matrix approved by product/legal.
2. Moderation visibility rules aligned across API + frontend behavior.
3. Legal request and DMCA handling runbooks (minimum launch version).

## Checklist
- [ ] Decide intended behavior for non-published/hidden/removed resources by endpoint.
- [ ] Confirm and align comment status visibility (`hidden` vs `removed`) in public responses.
- [ ] Define content license + moderation rights language inputs for ToS.
- [ ] Create legal request response workflow (owner, intake channel, response standard).
- [ ] Create DMCA/copyright takedown workflow (owner, intake, turnaround, repeat-infringer handling).

## Exit Criteria
- Public behavior and legal text match for visibility/moderation.
- Legal request/DMCA workflows are documented and owned.
